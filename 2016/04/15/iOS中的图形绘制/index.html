1.iOS中如何把图形绘制到屏幕上

如果你想深入的了解iOS中的绘图，那么你先得了解iOS中几个用于绘图的框架以及他们的由来

CoreGraphics框架:

有时候CoreGrphics也会被称为Quartz 2D，但是我认为这是不准确的，我们来看苹果的官方文档介绍：

The Core Graphics framework is a C-based API that is based on the Quartz advanced drawing engine. It provides low-level, lightweight 2D rendering with unmatched output fidelity. You use this framework to handle path-based drawing, transformations, color management, offscreen rendering, patterns, gradients and shadings, image data management, image creation, masking, and PDF document creation, display, and parsing.

CoreGraphics框架是基于Quartz 2D的，并且UIKit框架和AppKit框架也包含了Quartz 2D的一些简单API，如UIBeizerPath就是UIKit框架下用于绘图的API。对于iOS来说，我们所有的图形绘制都是基于Quartz 2D的(后面解释为什么)，但是我们不能说图形绘制都是基于CoreGraphics的，这就是两者的区别所在。CoreGraphics框架下用于绘制图形的API是CGContextRef。

UIKit框架：

iOS里只要涉及显示的控件基本都是UIKit下的，而图形绘制的API-UIBeizerPath只是其中很小的一部分功能，这里注意下，我在前文中也有提到，UIBezierPath是基于Quartz 2D的。

QuartzCore框架

这个框架中用于绘图的API是CALayer，用于动画的API是CAAnimation。

所以在iOS中用于绘图的API有3个，他们分别是CGContextRef，UIBezierPath，和CALayer。CALayer在绘图的时候，它的底层也是通过CGContextRef实现的，所以，iOS的绘图都是基于Quartz 2D的。

绘图并不是随时调用都是有效的，比如你要画一个圆，你在ViewDidLoad中调用它，那么他是不会起作用的，iOS的图形绘制你需要把它加到当前的位图上去，并且在合适的时间调用它。

有3种方式

  • 自定义UIView
  • 自定义CALayer
  • 实现图层代理

2.实现绘图的几种方式

1.自定义UIView

要想你的绘制实现，你需要在自定义UIView对象的.m文件中重写drawRect:(CGRect)rect方法，在这个方法中的绘制会显示在屏幕上。

能触发视图更新的动作，也就是调用drawRect:(CGRect)rect方法的触发条件有：

  • 添加视图addSubView
  • 对遮挡您的视图的其它视图进行移动或删除操作
  • 将视图的hidden属性声明设置为NO，使其从隐藏状态变为可见
  • 将视图滚出屏幕，然后再重新回到屏幕上
  • 显式调用视图的setNeedsDisplay或者setNeedsDisplayInRect:方法
